01 — 市場の変化

患者行動は、
すでに変わり始めている

ホットペッパーやGoogleで検索する時代から、AIに話しかけて予約する時代へ。その移行は静かに、確実に始まっている。

01

患者は「検索」しなくなる

ChatGPTやGeminiに条件を話しかけると、AIが複数の医院を比較してひとつを推薦する。ホットペッパーやGoogleマップの順位を争う時代から、AIの推薦リストに載るか否かを争う時代へ。

02

AIは「情報が読めない医院」を除外する

診療メニューが曖昧なら除外。認定資格が確認できなければ除外。予約システムがAIと連携していなければ除外。患者は選ばなかったのではなく、最初から見せてもらえなかっただけだ。

03

予約システムの競争軸が変わる

これまでの競争軸は「WEB予約・LINE予約・リコール」だった。しかし次の競争軸は決まっている——外部AIが予約を入れられるか否か、だ。

近未来のシナリオ
渋谷で土曜も診てくれるインビザライン対応の歯医者、来週の午後で予約して。
条件に合う医院を3件見つけました。〇〇デンタルクリニックが最も口コミ評価が高く、来週土曜の14時に空きがあります。予約を確定しますか？
お願いします。
予約が確定しました。確認SMSをお送りしました。
⚠️ このシナリオを実現するには、予約システム側のReservation APIが必要。現在、外部APIを公式公開している歯科予約システムは日本にほぼ存在しない。

03 — 調査の意義

なぜ今、この調査をしているか

競争軸が変わる前に、現状を正確に把握することが業界全体の利益になる。

01

「AI予約に対応できるか」を整理した調査がない

WEB予約・LINE予約・リコール——これらでベンダーを比較する情報は多い。しかし「AI Agentが予約を入れられるか」という軸で市場を整理した調査は国内に存在しない。

02

MCP普及でゲームが変わる

Model Context Protocol（MCP）の普及により、2026〜2027年にかけてAIと外部サービスの連携が急加速する。準備していたベンダーと、していなかったベンダーの差は、この12ヶ月で決まる。

03

調査を通じて、業界の共通言語を作る

「AI Readiness」という概念が業界内に定着することが目的だ。ベンダー・医院・患者の三者が同じ言語で話せる状態を作ることで、市場全体の移行速度が上がる。

04 — Why Dental

なぜ歯科領域から始めるのか

AI検索から予約までの行動変化が、最も早く起きる可能性がある領域の一つ。それが歯科だ。

💰

自由診療の拡大で「比較検討」が長くなる

インビザライン・ホワイトニング・審美補綴——価格差が大きい自由診療は、患者の比較行動を誘発する。AI検索との相性が、医療分野の中で際立って高い。

🔍

「近くの歯医者」はAI検索の定番クエリになる

ChatGPT・Perplexity・Geminiで既に「おすすめの歯医者」を検索するユーザーが増えている。AIが推薦する医院と、されない医院の差は、今この瞬間も広がっている。

📅

予約DXの普及で「AI経由予約」の土台が整いつつある

WEB予約・LINE予約の普及が、次のステップ——AI Agentによる自動予約——の地ならしをしている。インフラが整った市場ほど、次の変化が早く来る。

🌐

海外ではすでに動いている

米国のFrontDeskIndexはAI予約対応に特化したSaaSランキングを運営し、英語圏の歯科SaaS市場では「AI Agent対応」が選定基準の一つになりつつある。

Evaluation Framework
Discovery Readiness
AIが医院・サービスを発見できなければ、推薦は発生しない。構造化データ・GEO対応・LLM引用状況を評価する。
Recommendation Readiness
AIが推薦するには、信頼できる根拠が必要になる。Entity最適化・引用実績・AI時代の集患方針を評価する。
Action Readiness
AIが予約・申込を実行するには、接続可能な導線が必要になる。Reservation API・MCP対応・外部AI連携実績を評価する。
本調査は歯科医院・予約システム事業者・AIプラットフォームの声をもとに継続更新されます。
05 — 調査設計

3つの軸で実態を把握する

Discovery・Recommendation・Actionの3フェーズに対応した評価軸で、各ベンダーのAI対応実態を測る。

🔍

GEO Readiness

→ Discovery：AIに発見されるための基盤
  • 構造化データ（JSON-LD）対応
  • 医院ページのEntity最適化支援
  • LLM引用最適化（AI検索対策）
  • ChatGPT/Perplexityでの引用状況
  • GBP・Merchant Center連携
  • AI時代の集患支援方針
🔗

Platform Readiness

→ Recommendation：AIに選ばれ続けるための信頼基盤
  • LINE公式アカウント連携深度
  • CRM・MA機能の充実度
  • 外部API・Webhook提供状況
  • SDK・開発者向けドキュメント
  • レセコン連携の幅と深度
  • 今後12ヶ月のロードマップ
🤖

AI Agent Readiness

→ Action：AIが実際に予約を完結できるか
  • Reservation API 公開有無
  • 外部AIからの予約登録可否
  • Schema.org ReservationAction対応
  • MCP対応の方針・ロードマップ
  • AI電話・ボイスボット連携実績
  • 外部AI連携に関する公式見解
