Agent Readiness Index
Methodology

AI Agent時代において、企業・サービスがどのように評価されるかを定義するフレームワーク。 調査の背景、評価軸、プロセスを公開します。

Why Agent Readiness Matters

検索行動は、
変化している

顧客が情報を得て、比較し、行動するプロセスが根本から変わりつつある。 これは仮説ではなく、すでに始まっている変化だ。

従来の顧客行動
🔍
検索 Google / ホットペッパーで調べる
👤
比較 人間が候補を読んで選ぶ
📅
予約 Webフォームで人が入力・完了
→
AI Agent時代の顧客行動
🤖
AI検索 ChatGPT / Perplexityに話しかける
🔄
AI推薦 Agentが候補を比較・選定する
✅
AI行動 Agentが予約・申込を代行・完結
企業やサービスは、Webサイトだけでなく、AI Agentからどのように認識され、推薦され、行動対象となるかを考える必要があります。 Agent Readiness Indexは、この変化を可視化するための研究プロジェクトです。
What We Measure

3つの観点から評価する

Agent Readiness Indexでは、企業・サービス・プラットフォームがAI Agent時代にどれだけ対応できているかを、 Discovery・Recommendation・Actionの3軸でスコアリングします。

01
🔍
Discovery Readiness
AIに発見される能力

AIに発見されなければ、推薦も行動も発生しません。 Agentが情報を取得・解析できる構造が整っているかを評価します。

  • Entity設計
  • 構造化データ（JSON-LD / Schema.org）
  • Knowledge Base整備
  • Citation・LLM引用状況
  • Third-party Presence
02
⭐
Recommendation Readiness
AIに推薦される能力

AIは単に情報を検索するだけでなく、信頼できる候補を推薦します。 推薦の根拠となる情報資産や専門性が重要になります。

  • Expertise（専門性の可視化）
  • Authority（権威性の裏付け）
  • Trust Signals（信頼シグナル）
  • Original Content
  • Reputation（評判情報）
03
⚡
Action Readiness
AIが行動できる能力

将来的にAI Agentが予約・申込・問い合わせを支援または代行するためには、 行動可能な導線が必要になります。

  • API公開状況
  • Authentication設計
  • Reservation Flow
  • Integration（外部連携）
  • Agent Compatibility（MCP等）
Research Process

調査プロセス

Agent Readiness Indexは、以下のプロセスで調査を実施します。 公開情報・実地検証・ヒアリングを組み合わせた多角的なアプローチです。

Step 1
公開情報調査

企業サイト、サービスサイト、技術資料、ドキュメントなどを調査します。AI Agentが参照しうる情報ソースを網羅的に収集・分析します。

Step 2
プロダクト分析

実際のサービスや導線を確認し、ユーザー体験や接続性を評価します。API・Webhook・MCP対応などの技術仕様を検証します。

Step 3
業界ヒアリング

事業者、サービス提供企業、業界関係者へのヒアリングを行います。公開情報では把握できない方針・ロードマップ・実態を収集します。

Step 4
レビュー・確認

調査内容の妥当性を確認します。掲載対象企業には、公開前にスコアと内容を確認・修正できる機会を提供します。一方的な掲載は行いません。

Step 5
公開・更新

レポート公開後も継続的に更新します。AI Agent市場は急速に変化しており、スコアは定期的に再評価・更新されます。
